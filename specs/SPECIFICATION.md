# FlowMaster - 테스트 결과 전자결재 시스템 스펙

## 1. 개요 (Overview)
*   **목적**: 기존 엑셀 파일로 관리하던 테스트 결과 및 이력을 DB 기반의 전자결재 시스템으로 전환하여 데이터 무결성을 확보하고 이력 관리를 체계화함.
*   **핵심 가치**: 
    *   테스트 결과의 체계적 기록 및 검색.
    *   전자결재를 통한 승인 절차 투명성 확보.
    *   Legacy 데이터(백업 DB) 활용성 증대.

## 2. 사용자 및 권한 (Users & Roles)
*   **인증 방식**: Active Directory (AD) 계정 연동.
*   **권한 체계**:
    1.  **일반 사용자 (General User)**: 
        *   테스트 결과 입력 및 저장.
        *   결재 기안(상신).
        *   본인 기안 문서 조회.
    2.  **결재권자 (Approver)**:
        *   기안된 문서 검토 및 승인/반려.
        *   (옵션) 코멘트 작성.
    3.  **관리자 (Admin)**:
        *   전체 문서 조회 및 관리.
        *   사용자 권한 관리 (필요 시).
        *   시스템 설정 관리.

## 3. 핵심 기능 (Core Features)
### 3.1 테스트 결과 관리
*   사용자는 수행한 테스트 결과를 입력 폼을 통해 작성.
*   작성된 데이터는 연결된 별도 DB 시스템에 저장.
*   **백업 DB 조회**: 다른 시스템에서 사용 후 백업된 DB 데이터도 불러와서 조회 가능해야 함.

### 3.2 전자결재 프로세스
*   **흐름**: 작성(임시저장) -> 기안 -> (검토) -> 승인/반려 -> 완료.
*   **알림**: 결재 상태 변경 시 대상자에게 통지.
    *   채널: MS Teams 메시지 및 이메일(AD 계정 기반).
    *   시점: 기안 도착 시(결재자), 승인/반려 완료 시(기안자).

### 3.3 기타
*   **플랫폼**: Windows Desktop Application (WPF 권장).
*   **확장성**: 추후 MS Teams 내에서도 결재 승인 처리가 가능하도록 고려 설계.

## 4. 기술 및 환경 제약 (constraints)
*   **개발 환경**: Visual Studio 2019 Pro (C#).
*   **데이터베이스**: 외부 기존 DB 시스템 활용.
*   **사용자 규모**: 약 15명 내외 (Low Traffic).
*   **운영 환경**: 사내 네트워크 (AD 환경).

## 5. UI/UX 컨셉
*   직관적인 데이터 입력 폼 (엑셀과 유사하거나 더 편리한 UI).
*   내 결재함 / 기안함 등 상태별 대시보드 제공.
